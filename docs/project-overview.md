# English Beater 项目总览

## 1. 这个项目是做什么的

`English Beater` 是一个“节奏驱动”的英语词汇学习应用。核心思路是把背单词和答题做成轻量游戏，通过连击、得分、鼓点反馈来提升学习的趣味性和持续性。

面向用户的主要体验是：
- 选择词汇难度后进行 3 轮答题；
- 每轮 10 题，包含单选和多选；
- 记录分数、连击、错词；
- 在词典页做收藏/加星/笔记，并查看学习进度。

## 2. 当前实现形态

项目是一个前后端分离的 TypeScript 工程：

- 前端：`frontend/`
  - React + Vite + Zustand + Ant Design + Tailwind
  - 负责登录页、主菜单、游戏页、结果页、词典页 UI 与交互
- 后端：`backend/`
  - Express + TypeScript
  - 提供登录、词库查询、题库生成、学习进度读写等 API
- 文档：`docs/`
  - 已有 `docs/api/backend-api.md`（后端接口说明）

## 3. 核心功能拆解

### 3.1 游戏化答题
- 支持 3 轮游戏流程（每轮 10 题）
- 题目由后端实时生成：
  - 前 8 题为单选（英文词 -> 选中文释义）
  - 后 2 题为多选（选出积极/中性的词）
- 答对增加连击和分数，答错会重置连击并记录错词
- 前端通过 Web Audio API 生成鼓点反馈（正确/错误音效不同）

### 3.2 词典与学习进度
- 词典页支持查看全量词汇及按等级/收藏状态筛选
- 支持对单词进行：
  - `starred`（加星）
  - `favourites`（收藏）
  - `notes`（笔记）
- 学习进度可通过后端 API 拉取与保存

### 3.3 登录与鉴权
- 前端启动后默认调用登录接口（开发态使用 `code=mock`）
- 后端返回 `openid + JWT token`
- 进度接口使用 `Authorization: Bearer <token>` 鉴权

## 4. 关键业务流

### 4.1 启动流
1. 前端 `App` 初始化调用 `initAuth`
2. 调用 `/api/auth/login` 获取 token
3. 调用 `/api/user/progress` 拉取用户学习数据
4. 进入菜单，开始游戏或进入词典

### 4.2 游戏流
1. 在主菜单选择词汇等级（CET-4 / CET-6 / IELTS）
2. 每轮调用 `/api/game/questions?level=...&round=...`
3. 前端判题并更新分数、连击、错词
4. 三轮结束进入结果页，展示分数、最大连击、错词列表

### 4.3 词典学习流
1. 拉取词库 `/api/dictionary`
2. 用户执行加星/收藏/笔记操作
3. 调用 `/api/user/progress` 持久化学习状态

## 5. 数据与题库来源

- 词库主数据位于 `backend/data/words.json`
- 如果词库不足，后端会尝试从 markdown 学习手册解析并回写 `words.json`
- 题库生成逻辑在 `backend/services/question.service.ts`

## 6. 当前项目特点与限制

### 已有优势
- 结构清晰，前后端职责边界明确
- API 协议统一，文档齐全（已有后端接口文档）
- 功能闭环完整（登录 -> 答题 -> 复盘 -> 词典进度）

### 当前限制
- 学习进度使用内存 `Map` 存储，服务重启后会丢失
- 登录页“账号密码/微信登录”在前端主要是演示态，真实校验有限
- 暂无自动化测试与持久化数据库层

## 7. 一句话总结

这是一个以“快乐学习”为导向的英语词汇游戏化应用原型，已经具备完整 MVP 闭环（鉴权、题库、答题、进度管理），适合继续向“真实用户可持续使用”的方向迭代（数据库持久化、真实登录、测试与部署完善）。
